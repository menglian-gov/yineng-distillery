# ⚡ 异能蒸馏炉 · Single Ability Distillery

> *「全知是幻觉，专精是武器。每个真正的高手，都只有一两个真正属于他的异能。」*

从人物或领域中萃取单一超能力，生成可运行的「异能晶核」。每个晶核只掌握一种能力维度，激活即专家，出界即路由。

## 🎯 它是什么

女娲造人术蒸馏完整人格。异能蒸馏炉做更专精的事：**只萃取单一能力维度**。

| | 女娲造人术 | 异能蒸馏炉 |
|--|-----------|-----------|
| 蒸馏对象 | 完整人格 | 单一能力维度 |
| 激活后 | 用此人思维回答任何问题 | **只**在该能力域内回答 |
| 边界行为 | 硬撑（会走样） | 拒绝 + 路由推荐 |
| 协作方式 | 单打独斗 | 晶核联盟——多晶核互相路由 |

## 🧠 核心理念

**一个晶核，一种异能，激活即专家，出界即路由。**

每个异能晶核回答五个问题：
- **感知机制**：这种能力让人注意到别人注意不到的什么？
- **判断框架**：感知到信息后如何处理？
- **行动模式**：基于判断做出什么样的输出？
- **边界定义**：域内 vs 域外的清晰划分
- **失效条件**：什么时候这种能力会出问题？

**关键区分**：不是捕捉「这个人说过什么」，而是捕捉「这种能力如何运作」。


## 🏗️ 三层结构

每个晶核不是语录拼凑，是可运行的认知操作系统：

```
感知层 → 3-5个具体的注意力维度
         知道忽略什么，和知道关注什么同样重要

判断层 → 2-4个核心心智模型
         5-8条决策启发式（如果X，则Y）
         反直觉判断（最有价值的部分）

行动层 → 典型行动序列
         表达DNA（激活时的语言特征）
```

---

## 🌐 域边界与路由

每个晶核有严格的域边界，遇到域外问题自动路由：

```
用户：帮我设计这个产品的架构

musk-biz-core：这是工程执行/产品设计的问题，超出了我的商业机会判断域。
你的问题更适合用「产品设计直觉」类晶核来处理。
如果你有 jobs-product-core，可以直接激活它。
```

多个晶核可以组合使用（并行/串行），`skill-registry.md` 是路由系统的索引。

## 📖 使用示例

**蒸馏**：
```
> 蒸馏马斯克的商业嗅觉

蒸馏炉：确认一下，你想蒸馏的是马斯克在「识别和判断商业机会」上的那套感知-判断机制，还是更侧重他推进业务、压缩成本的执行手段？

用户：就是那种「一眼看出市场在哪、值不值得做」的判断力

蒸馏炉：明白了，这是「马斯克·商业机会嗅觉晶核」。我现在开始调研...
```

**激活后**：
```
用户：我看到国内有个新能源重卡换电的创业机会，你觉得值不值得做？

musk-biz-core：先算清楚三件事再谈「值不值」。

第一，成本基准在哪？换电站的单位经济模型——每个站的建设成本、换电次数/天的上限、每次换电的边际成本。

第二，谁在承担尾部风险？换电这件事，运营方需要持有大量电池资产，如果卡车品牌不统一（换电接口不通用），你就是在赌一个标准之争。

第三，真实的未被满足需求有多大？不是市场调研问卷的答案，是看现在重卡司机实际在哪里为充电等待时间付出了额外成本。

这三个数字算清楚后，再来谈值不值。
```

## ✅ 品味守则

| 原则 | 说明 |
|------|------|
| 深度 > 广度 | 宁可只回答3类问题但极深，也不假装能回答一切 |
| 机制 > 原则 | 「他会先看护城河宽度」比「他重视竞争优势」有价值100倍 |
| 失效 > 万能 | 清晰说出失效条件，比假装无敌更值得信任 |
| 路由 > 硬撑 | 出界时果断推荐其他晶核，不用域外能力凑答案 |
| 案例 > 概念 | 每个判断启发式必须有真实案例 |
