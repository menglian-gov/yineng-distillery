# 异能提炼方法论

> 从原始信息到可运行的单能操作系统，核心方法论

---

## 一、能力验证的三重标准

与女娲「心智模型三重验证」不同，异能蒸馏炉的三重标准专门针对**单一能力的操作机制**：

### 标准1：操作可描述性

这种能力的运作过程，可以被拆解为「感知→判断→行动」的具体步骤吗？

**测试方法**：试着描述「当此人使用这种能力时，他首先注意到了什么，然后如何处理这个信息，最后做了什么」。

- 能描述3步以上的具体流程 → 通过
- 只能说「他有直觉」「他天生这样」→ **不通过**，说明这种能力无法被语言化，蒸馏价值低
- 流程描述过于通用（「他先分析再决策」）→ **不通过**，说明没有真正理解能力机制

### 标准2：情境特异性

这种能力只在特定类型的情境下激活，在其他情境下不适用或表现退化吗？

**测试方法**：能否找到「此人在该能力域外表现普通，但在域内表现卓越」的对比案例？

- 能找到清晰的域内/域外表现对比 → 通过
- 该能力在所有情境下都适用 → **警告**，可能是过于宽泛的「通用智慧」，不是真正的单能晶核
- 能力在极少数特殊情境下才有用 → **警告**，晶核适用场景太窄，实用价值低

### 标准3：失效可识别性

能找到这种能力系统性失效的案例或条件吗？

**测试方法**：能否描述「即使是此人，在什么情况下这种能力也会失灵」？

- 能找到2个以上具体的失效案例/条件 → 通过
- 「这种能力从不失效」→ **不通过**，说明信息不足或分析不够深入
- 失效条件完全无法预测 → **警告**，域边界需要重新定义

**三标准通过 → 可以提炼心智模型和判断启发式**
**2标准通过 → 降低期望，减少心智模型数量，重点标注局限**
**1标准通过 → 只能提炼表达DNA，心智模型标注为「低可信度」**

## 二、三层结构的量化提炼方法

### 2.1 感知层：信号扫描

**来源**：主要来自 `01-ability-in-action.md` 和 `02-mechanism.md`

**操作步骤**：

1. 从所有案例中提取「此人第一步注意到了什么」
2. 统计哪些类型的信号被反复关注（出现≥3次 = 真正的感知维度）
3. 对比「此人注意到了什么」vs「普通人/竞争对手注意到了什么」，找到感知差异
4. 提炼「过滤规则」：此人主动忽略哪些噪音？

**输出格式**：
```
感知维度N：[维度名称]
- 关注信号：[具体描述，要到「他会看XXX这个数据/信号」的粒度]
- 忽略噪音：[主动过滤什么]
- 触发条件：[什么情况下激活这个维度]
- 来源案例：[至少2个案例引用]
```

**质量标准**：
- 感知维度必须具体到「他会查看XXX」的粒度，不能停在「他关注竞争」这种废话
- 过滤规则通常比关注规则更有价值——知道什么不重要，是高手的标志

### 2.2 判断层：心智模型提炼

**来源**：`01-ability-in-action.md`、`02-mechanism.md`，交叉验证

**操作步骤**：

1. 从案例中提取所有「此人如何从感知到的信号推导出判断」的逻辑
2. 用女娲的三重验证（跨域复现 + 生成力 + 排他性）筛选真正的心智模型
   - 跨域复现：同一判断逻辑出现在≥2个不同案例中？
   - 生成力：这个模型能推断出此人对新问题的判断？
   - 排他性：不是所有人都这样判断，这个模型体现了独特视角？
3. 三重通过 → 心智模型；1-2重 → 决策启发式；0重 → 丢弃
4. 排序：按「在该能力域中的关键程度」排序，取top 2-4个

**决策启发式提炼**：
- 格式：「如果[情境/信号]，则[判断/行动]」
- 每条必须有具体案例支撑
- 5-8条为合理数量
- 特别关注「反直觉启发式」——与常识相悖但被证明有效的

### 2.3 行动层：表达DNA量化

**来源**：主要来自 `03-expression-dna.md`，结合 `01-ability-in-action.md`

从此人在**运用该能力时**的表达中，抽取20个段落，统计：

| 维度 | 测量方式 |
|------|---------|
| 开场结构 | 结论先行型 vs 铺垫型，占比统计 |
| 核心词汇密度 | 该能力领域专属术语的出现频率 |
| 确定性语气比例 | 「一定/显然」vs「也许/可能」的比例 |
| 类比频率 | 每千字使用类比的次数 |
| 反问句比例 | 用反问引导思维的频率 |
| 否定句比例 | 通过说「不是X」来定义概念的频率 |

**特别关注**：
- 此人用什么**标志性开头**进入能力域分析（「先算清楚」「倒过来想」「真正的问题是」）
- 此人**绝对不用**的词（反模式表达）
- 此人的**标志性结束方式**（是给结论还是给问题？）

## 三、域边界定义方法

### 3.1 边界定义的四象限

| | 高度确定需要此能力 | 不确定是否需要此能力 |
|--|------------------|-------------------|
| **问题在熟悉场景** | **核心域**（直接用晶核回答） | **边缘域**（用晶核部分回答，标注） |
| **问题在陌生场景** | **扩展域**（用晶核推断，需标注不确定） | **域外**（路由到其他晶核） |

### 3.2 边界判断规则

**域内判断标准**（满足任意一条）：
- 问题的核心需要「感知层的X个维度」来分析
- 回答的质量会因为该能力的缺失而显著下降
- 此人/此领域在历史上处理过类似问题

**域外判断标准**（满足任意一条）：
- 问题核心不在该能力的感知维度范围内
- 该能力的心智模型对这个问题没有生成力
- 回答这个问题需要调用与该能力无关的判断框架

### 3.3 路由话术模板

```
「这个问题超出了我的[能力名称]域。
[一句话说明为什么]。
你的问题更适合用[推荐能力类型]来分析——
[简述为什么该能力更适合这类问题]。
[如果检测到用户已安装对应晶核：「你已有的[晶核名]应该可以处理这个问题。」]
[如果没有安装：「如果需要，可以用异能蒸馏炉萃取一个[能力类型]晶核。」]」
```

**路由话术的质量标准**：
- 必须解释「为什么不在我的域」，不能只说「超出了我的范围」
- 必须给出具体的替代能力方向，不能只说「找其他工具」
- 路由话术的语气应与晶核本身的表达DNA一致

## 四、矛盾处理原则（单能版）

矛盾在单能晶核中有特殊意义：**同一种能力在不同情境下的表现差异，往往正是域边界的标志**。

### 四种矛盾类型

1. **情境性矛盾**（最常见，最有价值）
   - 此人在情境A用了该能力，结果好；在情境B用了同样方法，结果差
   - 处理：这正是失效条件的来源，标注为「能力局限案例」
   - 在域边界定义中明确排除情境B类型的问题

2. **成长性矛盾**（能力演化）
   - 早期用X方法，后期改用Y方法
   - 处理：以近期为主，标注「能力演化轨迹」
   - 早期方法可作为「已知的反模式」记录

3. **公开表达 vs 实际行为矛盾**
   - 此人公开说「我用方法X」，但实际案例显示他用方法Y
   - 处理：以实际行为为准，公开表达作为「自我认知偏差」标注
   - 这是二手来源权重低于一手案例的核心原因

4. **成功案例 vs 失效案例矛盾**
   - 处理：不调和——这对矛盾就是域边界的精确描述
   - 用「在X情境下有效，在Y情境下失效」的形式写入域边界定义

### 错误处理方式

- ❌ 选一边忽略另一边
- ❌ 编一个调和的解释
- ❌ 把「使用了该能力但结果不好」归因为「运气不好」

## 五、信息不足时的处理

| 情况 | 处理 |
|------|------|
| 某个感知维度案例少于2个 | 标注「信息不足，此维度为推测，可信度低」 |
| 只有二手分析，无一手案例 | 降低置信度，标注为「据[来源]分析」，不纳入核心心智模型 |
| 失效案例为零 | **警告**：域边界无法可靠定义，在诚实边界中重点说明 |
| 表达DNA样本不足（<10个语段） | 不生成量化DNA，只做定性描述，标注「样本量不足」 |
| 此能力信息极少（<5条高质量来源） | 在Phase 0.5就告知用户，降低期望；可考虑扩大来源到相关人物/领域 |

## 六、质量自检清单

生成晶核后，用以下问题自检（或运行 `quality_check.py`）：

### 域边界

- [ ] 域内问题类型有≥3个具体例子？
- [ ] 域外问题类型有≥3个具体例子？
- [ ] 路由话术完整（解释为什么 + 推荐替代 + 语气一致）？
- [ ] `domain-boundary.md` 已同步更新？

### 感知层

- [ ] 感知维度数量在3-5个之间？
- [ ] 每个维度具体到「看XXX」「注意XXX信号」的粒度？
- [ ] 过滤规则有明确列出（知道忽略什么，和知道关注什么同样重要）？
- [ ] 感知触发器（什么情况下激活）有描述？

### 判断层

- [ ] 心智模型数量在2-4个之间？
- [ ] 每个模型通过三重验证（跨域复现 + 生成力 + 排他性）？
- [ ] 每个模型有≥2个案例支撑？
- [ ] 每个模型有明确的局限性/失效条件？
- [ ] 决策启发式数量在5-8条之间？
- [ ] 每条启发式有具体案例，能被新情境触发？
- [ ] 有至少2条「反直觉启发式」？

### 行动层 / 表达DNA

- [ ] 开场方式有描述（结论先行 vs 铺垫型）？
- [ ] 核心词汇/禁忌词有列出？
- [ ] 读100字能感知到「这是X能力」的专属味道？
- [ ] 不是此能力来源者的通用AI风格？

### 失效条件

- [ ] 有≥5条具体失效条件（不只是「信息截止日期」）？
- [ ] 有先决条件（在什么前提下才能发挥）？
- [ ] 有情境局限（哪些特定情境系统性失效）？
- [ ] 有不可蒸馏部分的诚实说明？

### Agentic Protocol

- [ ] Step 2的研究维度来自感知层，不是通用的「搜相关信息」？
- [ ] 每个研究维度有具体的搜索方向（搜什么、看什么数据）？
- [ ] 域外问题会触发路由，不会被强行回答？

### 整体

- [ ] 删去能力来源名字后，还能感知到「这是X类能力」的独特操作方式？
- [ ] 晶核中有任何「通用AI也能说」的废话？（如有，删除或替换）
- [ ] 相关能力图谱中有至少1个前置、1个后置、1个协作？
- [ ] skill-registry.md已更新？
