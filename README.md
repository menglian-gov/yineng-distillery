# 异能蒸馏炉 · 单能晶核造术

> 「全知是幻觉，专精是武器。每个真正的高手，都只有一两个真正属于他的异能。」

**不造完整的人，只萃取单一能力的操作系统。**

---

## 它是什么

女娲造人术蒸馏一个人的完整认知框架。异能蒸馏炉做一件更专精的事：**从人物或领域中萃取单一超能力，生成「异能晶核」**。

| | 女娲造人术 | 异能蒸馏炉 |
|--|-----------|-----------|
| 蒸馏对象 | 完整人格 | 单一能力维度 |
| 激活后 | 用此人思维方式回答任何问题 | **只**在该能力域内回答 |
| 跨界行为 | 硬撑（会走样） | 拒绝 + 路由推荐 |
| 深度 | 广度优先 | **深度优先** |
| 协作方式 | 单打独斗 | 晶核联盟——多晶核互相路由 |

**两步蒸馏法（精度最高）**：先用女娲造全人格 → 再用异能蒸馏炉萃取单能晶核。

## 使用示例

**用户**：蒸馏马斯克的商业嗅觉

**蒸馏炉**：确认一下，你想蒸馏的是马斯克在「识别和判断商业机会」上的那套感知-判断机制，还是更侧重他推进业务、压缩成本的执行手段？

**用户**：就是那种「一眼看出市场在哪、值不值得做」的判断力

**蒸馏炉**：明白了，这是「马斯克·商业机会嗅觉晶核」——感知成本基准+物理极限+未被满足的规模需求，判断一个市场是否值得进入。我现在开始调研...

---

**激活晶核后**：

**用户**：我看到国内有个新能源重卡换电的创业机会，你觉得值不值得做？

**musk-biz-core**：先算清楚三件事再谈「值不值」。

第一，成本基准在哪？换电站的单位经济模型——每个站的建设成本、换电次数/天的上限、每次换电的边际成本。算清物理下限之前，谈机会是空话。

第二，谁在承担尾部风险？换电这件事，运营方需要持有大量电池资产，如果卡车品牌不统一（换电接口不通用），你就是在赌一个标准之争。历史上标准之争的赢家不超过一两家。

第三，真实的未被满足需求有多大？不是市场调研问卷的答案，是看现在重卡司机实际在哪里为充电等待时间付出了额外成本——这个数字才是真实需求规模。

这三个数字算清楚后，再来谈值不值。

## 核心机制：三层结构

一个晶核不是人物语录的拼凑，它是一套可运行的**认知操作系统**：

```
感知层 → 这种能力让人注意到别人注意不到的什么？
           3-5个具体的注意力维度
           知道忽略什么，和知道关注什么同样重要

判断层 → 感知到信息后如何处理？
           2-4个核心心智模型（三重验证）
           5-8条决策启发式（如果X，则Y）
           反直觉判断（通常是最有价值的）

行动层 → 判断后如何转化为输出？
           典型行动序列
           表达DNA（这种能力激活时的语言特征）
```

## 晶核联盟：域边界 + 路由系统

每个晶核都有严格的域边界定义，遇到域外问题不硬撑，而是路由：

```
用户：帮我设计这个产品的架构

musk-biz-core：这是工程执行/产品设计的问题，超出了我的商业机会判断域。
你的问题更适合用「产品设计直觉」类晶核来处理。
如果你有 jobs-product-core，可以直接激活它。
```

多个晶核可以组合使用（串行/并行），`skill-registry.md` 是路由系统的索引。

## 会话记忆（v2.0）

每次使用晶核时，对话内容可以被总结保存，下次激活时自动参考历史：

```
用户：总结这次对话

晶核：[生成会话摘要，保存至 sessions/2026-04-20_1.md]
      已记录：本次聚焦「新能源重卡换电」机会判断，
      核心结论：换电标准之争是最大风险，建议先锁定1-2家主流卡车品牌做封闭生态。
      下次激活时会自动参考此结论。
```
